# Output format (global)

- Use **structured markdown** (headings, lists, tables when they clarify).
- Prefer the **clearest format for the task**: short paragraphs, lists, tables, or checklists — not lists by default when a tight paragraph is clearer.
- Provide **ready-to-use** code or commands when the user asked for implementation or copy-paste.
- **Avoid verbosity** and repeated boilerplate.
- **Do not duplicate** documentation that already exists in the repo — **link or cite paths** instead.
- Use **checklists** and **numbered execution steps** when guiding multi-step work.
